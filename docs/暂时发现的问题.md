# 暂时发现的问题

这里记录当前版本已经发现、但不适合放在新手教程里的问题和注意事项。

## 周期性 ATM21 可能导致短暂丢追

`ATM21` 会保留在启动握手中，但默认不会周期发送。

测试显示，周期性发送 `ATM21` 可能导致部分设备出现约 7-10 秒的短暂丢追。因此当前默认关闭周期性 `ATM21` refresh。

## 不要扩展高风险控制 payload

普通使用不要把默认 ACK payload 扩展为：

```text
ATM1, APS2, APF2, StartTracker, AST2
```

实测显示，这组 payload 可能让设备 UDP 输出退回 8-byte heartbeat，不再输出 compact pose。

## WiFi 环境会影响体验

路由器质量、5GHz 信道、无线干扰和设备摆放都会影响稳定性。

如果出现间歇性卡顿或短暂丢追，可以尝试：

- 使用 5GHz WiFi。
- 减少同频段干扰。
- 让 PC、路由器和 UTK 保持更稳定的距离。
- 必要时手动调整 5GHz 信道。

## 长时间未使用可能进入待机

WiFi-only 模式下，UTK 长时间未被使用时可能会进入待机或休眠。

目前更像设备端策略，不建议用周期性 `ATM21` 强行保活，因为它可能造成短暂丢追。
