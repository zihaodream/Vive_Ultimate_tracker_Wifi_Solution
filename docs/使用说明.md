# UTK WiFi Bridge 使用说明

本文面向普通测试和日常使用。Ultimate Tracker 后续简称为 UTK。

## 初始化准备

1. 建议 UTK 使用 169 系统版本。本项目基于该版本开发和测试，其他系统版本可能出现未知问题；经测试 168、169 版本均可使用，178 版本还未测试。
2. 在使用本 WiFi-only 方案前，一定要先使用官方接收器方案和 VIVE Hub 为 UTK 完成地图扫描。

3.首次配置需要 ADB环境（需先给自己电脑安装adb环境 中文教程 https://blog.csdn.net/QXXXD/article/details/151830144 英文教程：https://www.xda-developers.com/install-adb-windows-macos-linux/ Part：Add ADB to your Path environment variables ）。

## 推荐入口

优先使用 GUI：

```text
app\UTK-WiFiOnly-GUI.cmd
```

这个入口会打开图形操作界面，用于初始化 PC IP、启动服务和写入 UTK WiFi-only 配置。

## 推荐网络

推荐链路：

```text
UTK -> 5GHz WiFi router -> PC -> SteamVR
```

建议：

- 使用 5GHz WiFi。
- PC 和 UTK 保持在同一局域网。
- PC 使用固定局域网 IP，避免每次 DHCP 变化后都要重新写入 UTK。

## 界面按钮

`Initialize`

读取当前 PC 局域网 IP，填入 `Local PC IP`，并尝试请求管理员权限将当前网卡配置为静态 IP。

`Start Service`

启动 keepalive / pose bridge，自动注册 SteamVR OpenVR driver，并准备接收 UTK 的 WiFi-only 姿态数据。

`Flash WiFi`

通过 ADB 写入 UTK 的 WiFi-only 配置。只有首次配置、WiFi 信息变化、PC IP 变化或设备恢复后才需要重新点击。

`Status`

查看当前服务状态、端口监听和 Tracker 连接信息。

`Stop`

停止 PC 端测试服务。

`Debug mode`

仅用于抓包、排查和逆向分析。普通 VR 体验测试不要打开，它会增加日志量和运行负载。

`Device`

显示已连接 Tracker 的数量和对应 IP。其中 `Power off selected` 用于关闭指定设备，`Power off all` 用于关闭所有已连接设备。

`Dongle Mode`

将设备切换回官方接收器模式。通常在需要使用官方软件 VIVE Hub 更新地图时使用。

## 第一次使用流程

1. 准备一个 5GHz WiFi 路由器。
2. 让 PC 连接到该路由器。
3. 双击 `app\UTK-WiFiOnly-GUI.cmd`。
4. 在界面中点击 `Initialize`，读取当前 PC 局域网 IP，并尝试将该 IP 固定到当前网卡。
5. 填入路由器的 WiFi 名称和密码。
6. 确认 `Local PC IP` 是 PC 在该路由器下的局域网 IP。
7. 如果是第一次让 SteamVR 加载本项目驱动，建议先关闭 SteamVR。
8. 点击 `Start Service`，启动 PC 端 keepalive / pose bridge，并注册 SteamVR driver。
9. 如果 UTK 还没有写入 WiFi-only 参数，连接 ADB 后点击 `Flash WiFi`。
10. 启动 SteamVR，确认 Tracker 是否出现。

日常使用时，通常只需要：

1. 双击 `app\UTK-WiFiOnly-GUI.cmd`。
2. 在 GUI 中点击 `Start Service`。
3. 启动 UTK。
4. 观察 UTK 灯态：
   - 蓝灯闪烁：还未连接。
   - 呼吸绿灯：已经连接，但当前处于丢追状态。
   - 双闪绿灯：正常运行。
5. 启动 SteamVR。

只有在 WiFi 名称、密码、PC IP 改变，或者设备恢复到官方接收器模式后，才需要重新执行 `Flash WiFi`。

## 常见问题

### 是否需要官方接收器？

实时追踪链路不需要官方接收器。但首次使用必须先通过 VIVE Hub 和官方接收器完成匹配、连接和地图扫描。

### 是否每次都要 Flash WiFi？

不需要。只有首次配置、WiFi 信息变化、PC IP 变化或设备恢复后才需要重新写入。

### 为什么推荐固定 PC IP？

UTK 的 WiFi-only 配置中会写入 PC IP。若路由器给 PC 分配了新的 IP，UTK 仍会尝试连接旧 IP，导致连接失败。

### 可以突破 5 点上限吗？

本项目的目标之一就是让多台 UTK 通过普通 WiFi 路由器进入 SteamVR。当前 driver 默认保留 10 个 Tracker slot，内部支持更多 slot 的扩展空间。

### 使用 Flash WiFi 写入后，设备无法正常连接到电脑怎么办？

1. 检查 WiFi 名称和密码是否输入错误。如果有错误，输入正确内容后重新执行 `Flash WiFi`。
2. 如果 WiFi 名称和密码无误，可能是写入过程异常。可以使用 `Dongle Mode` 清除 WiFi-only 配置，并切换回官方接收器模式。
3. 如果仍然无法恢复，请使用 `APQ8053_ROM_FB.zip` 刷机包恢复系统。

### 如果需要更新地图（重新扫描地图）怎么办？

使用 `Dongle Mode` 切换回官方接收器模式，然后通过 VIVE Hub 更新地图。地图更新完成后，重新使用 `Flash WiFi` 写回 WiFi-only 模式即可。

## 当前运行链路

当前默认链路：

```text
UTK compact UDP 0x25
  -> PC UDP 9005
  -> Python keepalive / pose bridge
  -> binary UTKP 127.0.0.1:5557+
  -> SteamVR OpenVR driver
  -> SteamVR Tracker
```

默认运行策略：

```text
UdpPosePort:       9005
AckOnConnect:      true
ForwardBurstMode:  paced
PacedTargetHz:     50
PacedMaxDelayMs:   45
PoseForwardFormat: binary
ControlRefresh:    disabled
```

## 空间校准

UTK 进入 SteamVR 后仍需要做空间校准。

可使用开源校准项目：

[hyblocker/OpenVR-SpaceCalibrator](https://github.com/hyblocker/OpenVR-SpaceCalibrator)

## 刷机包使用说明

如果需要刷入或恢复 UTK 169 系统版本，请从本项目 GitHub Release 附件下载刷机包，并自行准备电脑驱动。

Release 附件文件名：

```text
APQ8053_ROM_FB.zip
```

本地整理目录中的刷机包位置：

```text
firmware_package\APQ8053_ROM_FB.zip
```

该目录默认被 `.gitignore` 忽略，不会提交到源码仓库。发布时请将 `APQ8053_ROM_FB.zip` 作为 GitHub Release 附件上传。

电脑需要具备：

```text
ADB 驱动
bootloader / fastboot 驱动
```

不同电脑和系统环境的驱动安装方式可能不同，请自行搜索并安装。刷机前请确认设备在 ADB 模式和 bootloader / fastboot 模式下都能被电脑正常识别。

基本流程：

1. 从本项目 GitHub Release 附件下载 `APQ8053_ROM_FB.zip`，或使用本地 `firmware_package\APQ8053_ROM_FB.zip`。
2. 将刷机包压缩文件解压到一个英文路径目录。
3. 将 UTK 连接到电脑。
4. 检查设备驱动是否正常，确认 ADB / bootloader 设备能被识别。
5. 在解压后的刷机包目录中运行：

```text
apq8053_flash_rom.bat
```

6. 等待脚本刷写完成。
7. 刷写完成后，按刷机包提示重启设备。

注意：

- 刷机有风险，可能导致设备无法启动、无法连接、数据丢失或设备损坏。
- 不要在刷机过程中断开 USB、关闭电脑或强制结束脚本。
- 请确认下载的是本项目说明中对应 UTK 169 的刷机包。
- 刷机相关问题不属于本项目自动处理范围。
